# SKILLS.md — ルートディレクトリ

この階層では、リポジトリ全体の構成と運用に関わる作業を行います。

## このディレクトリで得意なこと

- `README.md` の整備・更新（手順・モデル名・環境変数の整合を取る）
- `docker-compose.yml` を読み解き、コンテナ構成や依存関係を把握する
- `.env` / `.env.example` の項目とアプリコードの参照箇所を突き合わせる
- 開発者・エージェント向けドキュメント（`AGENTS.md` / `SKILLS.md`）の更新

## 作業上の注意

- 実際の秘密情報（トークン・パスワード）は絶対に新規記載しない。
- 既存の実装（`app/`）と README の記述が矛盾しないよう、変更時は両方確認する。
- `docker-compose.yml` は、サービス名・ポート・ボリュームマウントの意味を理解したうえで、最小限の変更に留める。
